# Flow Spec: oracle-schema-annotations

## 1.0 Context

Oracle AI Database 26ai schema annotations let applications store
database-resident metadata on schema objects. Cymbal Coffee already uses modern
Oracle DDL (`BOOLEAN`, `JSON`, `VECTOR(3072, FLOAT32)`, HNSW INMEMORY vector
indexes) and has extensive `COMMENT ON` metadata in the baseline migration.

This flow adds annotations to demonstrate the 26ai metadata surface while
preserving the current application runtime.

## 2.0 Requirements

- Add schema annotations to the baseline `0001` migration.
- Demonstrate annotations inline on tables, columns, and supported normal
  indexes.
- Preserve existing comments for compatibility.
- Document how to inspect annotations through Oracle dictionary views.
- Verify against Oracle 26ai, not only against parser behavior.

## 3.0 Implementation Plan

### Phase 1: DDL Annotation Contract

- [x] 1.1 Add inline annotation DDL to `src/app/db/migrations/0001_cymball_coffee_products.sql`.
  - Update the existing baseline migration in place. Do not create a new
    migration file.
  - Put `ANNOTATIONS(...)` clauses directly in the existing `CREATE TABLE`,
    column-definition, and supported normal index-definition DDL.
  - Use the Oracle inline shape, for example:
    ```sql
    CREATE TABLE product (
        id NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY PRIMARY KEY,
        name VARCHAR2(255) NOT NULL,
        in_stock BOOLEAN DEFAULT TRUE
            ANNOTATIONS (
                Display 'Global stock fallback',
                Data_Role 'availability'
            ),
        metadata JSON,
        embedding VECTOR(3072, FLOAT32)
            ANNOTATIONS (
                Display 'Product embedding',
                Embedding_Model 'gemini-embedding-2',
                Embedding_Dimensions '3072',
                Embedding_Purpose 'document',
                Distance 'COSINE'
            ),
        created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP WITH TIME ZONE DEFAULT ON NULL FOR INSERT AND UPDATE CURRENT_TIMESTAMP NOT NULL
    ) INMEMORY PRIORITY HIGH
      ANNOTATIONS (
          Display 'Cymbal Coffee products',
          Description 'Menu catalog rows with Vertex embeddings for semantic search',
          AI_Surface 'PRODUCT_RAG'
      );
    ```
  - Add `docs:start-schema-annotations` / `docs:end-schema-annotations`
    anchors around the representative inline block for docs literalinclude.

- [x] 1.2 Annotate the high-value schema objects inline.
  - `product`: product RAG/menu catalog purpose.
  - `product.embedding`: `gemini-embedding-2`, `3072`, document-purpose
    embedding input, `COSINE`.
  - `product.in_stock`: global fallback stock flag.
  - `store`: seeded cafe locations.
  - `store.latitude` and `store.longitude`: seeded store coordinates only;
    browser coordinates are request-scoped and not persisted.
  - `store.google_place_id`: optional place identifier; no-key Maps URLs remain

  - `response_cache`: cache scope and no coordinate persistence.
  - `embedding_cache`: Vertex embedding cache role and vector shape.
  - `search_metric`: telemetry role; avoid implying raw user location storage.
  - `product_category_idx`, `product_in_stock_idx`, `store_location_idx`, and

    describe model/dimension/distance metadata on the vector columns. The
    Oracle 26ai `CREATE VECTOR INDEX` page does not document an
    `annotations_clause`, so do not invent one without runtime evidence.

## 4.0 Known Risks

- Oracle syntax is authoritative. SQLSpec executes migration SQL through
  `execute_script`, so unsupported parser modeling does not justify moving
  annotations out of the inline DDL. The implementation must still validate on
  Oracle 26ai.
- Annotation values are limited to 4000 characters, so keep values short.
- Annotation names are identifiers. Quote names only when needed.
- Over-annotating ordinary columns can obscure the teaching point; focus on
  demo-critical semantics.
