# Product Requirements Document: Agent UI Response Speed Optimization

**Requirement Slug:** `agent-ui-update`
**Status:** Active
**Created:** 2025-10-24
**Agent:** Planner
**Implementation Agent:** Expert

## Executive Summary

Transform the Coffee Chat agent UI from a blocking synchronous pattern to an asynchronous streaming pattern. This is a **clean cut implementation** with no fallback mechanisms, providing immediate user feedback and real-time progressive AI responses.

## Problem Statement

Current UX has significant visual latency between form submission and response rendering:
- User message appears only after full server processing (~2000ms)
- No immediate feedback or loading indicators
- Agent response appears only after complete processing
- Blocking synchronous pattern creates perceived sluggishness

## Goals

### Primary Goals
1. **Instant User Feedback**: User message displays immediately (<100ms)
2. **Progressive AI Response**: Stream AI text as generated by ADK
3. **Maintained Performance**: Preserve existing cache hit rates (>80%)
4. **Enhanced UX**: Show loading states and real-time progress

### Success Metrics
- Time to First Byte (User Message): <100ms (baseline: ~agent_processing_ms)
- Time to First AI Token: <500ms
- Perceived Latency Reduction: 50%+
- Cache Hit Rate: Maintain ≥80%
- Streaming Overhead: <50ms
- SSE Connection Success Rate: >95%
- Zero Functionality Regression

## Technical Architecture

### Current Flow (Synchronous - TO BE REPLACED)
```
User Submit → Server Blocks (ADK Processing) → Complete Response → DOM Swap
             |------------ ~2000ms latency ------------|
```

### New Flow (Asynchronous - CLEAN REPLACEMENT)
```
User Submit → User Message Display → SSE Stream Connection → Progressive AI Text
   |<100ms|                        |<500ms to first token|
```

### Key Components
1. **Optimistic UI**: User message displayed before processing
2. **Server-Sent Events (SSE)**: Real-time event stream for AI response
3. **HTMX Out-of-Band Swaps**: Multiple DOM updates from single response
4. **Query State Cache**: Temporary storage (5 min TTL) for streaming context
5. **ADK Stream Adapter**: Convert ADK events to SSE format

### Technology Stack
- **HTMX 1.9.10+**: Already installed
- **HTMX SSE Extension**: MUST ADD (mandatory, not optional)
- **Litestar Stream**: Server-sent events support
- **Google ADK**: Native async streaming via `runner.run_async()`
- **CacheService**: Existing Oracle-backed cache

## Risk Assessment

### Technical Risks
1. **SSE Connection Drops**
   - Likelihood: Low
   - Impact: Medium
   - Mitigation: HTMX built-in auto-reconnect

2. **Increased Server Load**
   - Likelihood: Medium
   - Impact: Medium
   - Mitigation: Monitor concurrent connections, horizontal scaling

3. **Cache Consistency**
   - Likelihood: Low
   - Impact: Low
   - Mitigation: 5-minute TTL, automatic expiration

### UX Risks
1. **Confusing Partial Responses**
   - Likelihood: Low
   - Impact: Medium
   - Mitigation: Clear loading indicators, user testing
